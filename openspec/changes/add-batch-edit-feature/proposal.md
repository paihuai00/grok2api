## Why

当前图像编辑功能仅支持单张图片处理，用户需要批量处理多张图片时（如批量去背景、批量调色、批量添加效果），必须重复上传和操作，效率低下。添加批量编辑功能可以显著提升工作效率，满足批量处理场景需求。

## What Changes

- **新增"批量编辑"模式标签**：与现有的"生成"、"编辑"并列，作为第三种模式
- **支持多图片选择**：
  - 多选本地图片文件
  - 选择本地目录（批量导入目录内所有图片）
- **图片卡片选择机制**：
  - 每张图片卡片右上角添加选择框
  - 默认全选，未选中的卡片显示灰色
  - 标题右侧添加"全选"按钮
  - 若用户未选择任何图片，视为全选
- **批量处理执行**：
  - 优先并行处理，提升效率
  - 显示总体进度条
  - 失败的图片在专门区域显示错误信息
- **调用现有 API**：循环调用 `/v1/images/edits` 接口，无需后端改造

## Capabilities

### New Capabilities
- `batch-edit`: 批量图像编辑功能，包括多图片选择、目录导入、批量处理、进度显示、错误处理

## Impact

- **前端代码修改**：
  - `app/static/imagine/imagine.html` - 添加批量编辑 UI 元素
  - `app/static/imagine/imagine.js` - 添加批量编辑逻辑
  - `app/static/imagine/imagine.css` - 添加批量编辑样式
- **无 API 变更**：复用现有 `/v1/images/edits` 接口
- **无后端变更**：纯前端实现
- **浏览器兼容性**：目录选择功能仅支持 Chrome/Edge（File System Access API）
